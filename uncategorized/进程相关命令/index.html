Linux进程相关的命令

pstree

以树状显示进程的父子状态

ps

显示系统中的进程的详细信息

ps支持三个类型的选项

  • unix类型： -字母
  • BSD类型：字母
  • GNU类型：—单词

/proc下那些以数字为名称的目录的数字就是进程的信息映射

ps aux

a:显示现行终端下的所有进程的信息（包括其他用户的进程信息）

u:显示进程是哪个用户发起的

x:显示终端无关的进程信息

ps aux显示的信息：

USER：进程的属主

PID：

%CPU %MEM： 占用内存和空间CPU时间的百分比

VSZ： 每个进程的虚拟内存集大小

RSS：每个进程的常驻内存集大小

TTY：终端编号 ？表示终端无关编号

STAT 表示进程的状态

Here are the different values that the s, stat and state output specifiers (header "STAT" or "S") will display to
describe the state of a
process:
    D   uninterruptible sleep (usually IO)    //不可中断睡眠
    I   Idle kernel thread                    //闲置内核线程
    R   running or runnable (on run queue)     //运行或者可运行 
    S   interruptible sleep (waiting for an event to complete)   // 可中断睡眠
    T   stopped by job control signal                         //被作业控制信号停止
    t   stopped by debugger during the tracing                //在跟踪期间由调试器停止
    W   paging (not valid since the 2.6.xx kernel)            //内核2.6之后被启用
    X   dead (should never be seen)                           //死的进程
    Z   defunct ("zombie") process, terminated but not reaped by its parent    ////将要死的进程

For BSD formats and when the **stat** keyword is used, additional characters may be displayed: 

    <   high-priority (not nice to other users)    //高优先级
    N   low-priority (nice to other users)         //低优先级
    L   has pages locked into memory (for real-time and cust om IO)   //有页被锁住了
    s   is a session leader                        //会话进程的首进程
    l   is multi-threaded (using CLONE_THREAD, like NPTL pthreads do)    //多线程进程
    \+   is in the foreground process group               //前台进程组中的进程

START:进程的启动时间

TIME：进程在CPU上累计运行时间

COMMAND：执行哪个binging发起的这个进程

​ []：表示进程是由内核发起的进程

ps -elf

-e：显示所有进程（包括终端相关和无关），等同于ps -e = ps au

-l：以长格式显示

-f：显示进程的完整信息

ps -elf | head显示的信息

F：进程的标志位，通常为1和4,1表示fork之后未执行的进程，4表示root用户的进程

S：进程的状态

UID：进程的属主

PID：进程的编号

PPID：该进程的父进程的PID

C：该进程对CPU的百分率，以证书来显示

PRI：优先级，越小越高

NI：进程的nice值（默认nice值都是0）

ADDR：运行中的进程在内存中的地址编号

SZ：进程用掉的swap大小

WCHAN：表示进程是否处于运行状态，-表示正在运行

STIME：进程的发起时间

TTY：进程的终端编号，是否跟终端有关

TIME：实际占用CPU的时间

CMD：发起该进程的路径

ps -eFH

-F ：显示比小f更详细的信息

-H：以层级方式显示(树状图形式)

命令显示信息：

PSR：显示的是cpu的编号

显示指定用户的进程

-u username ：显示制定用户发起的进程

显示指定进程的PID值

-C：进程名

补充

o：指定要显示的字段部分，通常ax组合使用

例子：仅仅显示pid和command两个部分

命令：ps axo pid,command | head

pgrep和pkill

（显示的是静态信息）

pgrep：查找符合某个规则的进程（默认仅仅显示进程的ID号）

pkill ：结束符合某个规则的进程

选项：

  • -U uid|userName:指定真实用户（也就是且混用户前的那个用户）
  • -u uid|useName:指定运行进程的有效用户（当前运行进程的那个用户）
  • -t term：过滤指定终端相关的进程
  • -p ppid：过滤出以指定进程为父进程的子进程的列表
  • -l ：列举出进程的名称

例子：

pidof

（显示的是静态信息）显示指定进程的PID

top

（以动态方式，实时显示系统的进程，cpu，内存等信息）

默认：5s刷新一次

负载：cpu队列中等待使用cpu的任务个数

命令行选项：

  • -d ：指定刷新间隔
  • -p：仅显示指定id进程信息
  • -S：使用累加模式
  • -i：不显示空闲和僵尸状态的进程
  • -c：显示整个所执行的命令
  • -n num：指定刷新的次数

交互式命令：

  • l：显示/关闭第一行信息

htop

top命令的升级版本

  • Arrows：可以滚动显示
  • digits：可以按照pid数字查找
  • F3 /：
  • space（空格） ：标记选中的进程
  • U：取消所有标记的进程
  • F9或k：杀死某个进程
  • F7：高优先级，提高某个进程的优先级，减小nice值
  • F8：降低某个进程的优先级
  • a：设置cpu绑定，让某个进程工作在特定的cpu上
  • i：设置IO进程优先级
  • l ：显示进程所打开的文件
  • s：追踪进程所发起的进程调用
  • F2：设置
  • F1 ：打开帮助页面
  • F10或者q：退出
  • F5：树状来显示

vmstat

procs ：显示当前系统的队列信息

  • r:正在运行和等候运行进程的数量
  • b:表示的是阻塞的队列长度（其实就是等待IO完成的进程的个数，也就是处于不可中断状态的内存个数）

memory：表示内存相关的信息

  • swap：表示从物理内存移动到swap中的数据量
  • free：空闲物理内存大小
  • buff：缓存空间数据量（向磁盘进行写操作的内容）
  • cache：缓存空间的数据量（从磁盘进行读操作的内容）

swap：表示的是使用swap的速率信息

  • si：数据进入交互分区的速率，单位是kb/s
  • so：数据离开交互分区的速率，单位是kb/s

io：显示磁盘IO速率

  • bi：从内存向磁盘快写入数据的速率，单位是kb/s
  • bo：内存从磁盘块读取数据的速率，单位是kb/s

system：

  • in：系统每秒产生的中断数
  • cs（content Switch）：系统每秒进行多少次进程切换

dstat

dstat是vmstat的升级版本

选项：

  • -c：显示cpu的统计信息
  • -C cpu编号：显示具体的哪个CPU的信息
  • -d：只显示磁盘的统计信息
